📝 核心模块笔记：core/registry.py (自动注册中心)
📌 核心定位
系统的“户籍管理处”与“智能动态菜单”。它是实现系统高可扩展性（即插即用）的关键基石。
⚙️ 四大核心功能

实现“即插即用”的终极解耦（开闭原则）

痛点：传统写法在新增算法时，需要修改核心调度的 if...else... 代码，容易引发 Bug。

功能：提供 @register_algorithm 装饰器。新算法只需在 Wrapper 文件中打上此标签即可自动接入系统。对扩展开放，对修改封闭，永远不需要改动原有核心代码。

构建全局“算法字典”（精确派单员）

功能：在后端程序启动时，自动扫描并收集所有带有标签的算法，在内存中构建一个类似 {"DeblurDiff": deblur_wrapper_func} 的全局大字典。

效果：当 routes.py 需要执行某个算法时，直接通过字典的 Key（算法名）瞬间找到对应的 Wrapper 函数并执行，省去所有判断逻辑。

管理硬件约束元数据（掌管生死大权）

功能：装饰器不仅记录算法名字，还记录它的元数据（Metadata），比如 requires_gpu=True/False。

效果：为系统的“硬件约束感知”提供数据源。routes.py 在执行前会先查阅此记录，若环境不匹配则直接拦截，防止纯 CPU 环境下调用 CUDA 算子导致服务器崩溃。

为前端动态生成级联菜单（前台展示）

功能：向 API 提供当前所有可用算法的结构化列表（按“任务类型”分组，如曝光修正、去模糊）。

效果：前端无需硬编码写死下拉框，直接通过 API 获取并动态渲染菜单。后端加了新算法，前端页面刷新后自动多出一个选项。
